Scientific ML for Radiative Transfer

Problemulator

Problemulator is a radiative-transfer emulator that learns a mapping from one-dimensional atmospheric structure to layerwise flux outputs. The repository focuses on scientific correctness, strict configuration, and a reproducible preprocessing-to-training pipeline.

Overview

What this project models

The current objective is to emulate deterministic radiative transfer outputs from atmospheric profiles. Inputs include profile variables such as pressure and temperature, along with global context such as orbital distance.

Outputs are layerwise radiative quantities, including net thermal and reflected flux. The implementation is organized around fail-fast data validation, explicit normalization policy, and reproducible training.

Workflow

From raw profiles to trained checkpoints

1. Data contract

Raw HDF5 inputs are treated as the canonical source. Required keys, shapes, and finite values are validated before the pipeline proceeds.

2. Preprocessing

Dataset splits, normalization metadata, padding masks, and processed shards are prepared according to the checked-in configuration.

3. Training

The configured training pipeline uses explicit precision settings, deterministic seeding, and a transformer-based model trained on valid non-padding targets.

4. Evaluation

Training produces checkpoints, logs, metadata, and immediate held-out test metrics to summarize normalized-space model performance.

Repository

Where to look in the codebase

src/

Core pipeline logic for preprocessing, dataset loading, model definition, hardware setup, and training.

config/

Runtime configuration that defines data paths, normalization policy, precision, model hyperparameters, and training settings.

unit_tests/

Focused tests covering configuration behavior, dataset and model expectations, and training loop semantics.

Getting Started

Main entrypoint and default configuration

The configured pipeline entrypoint is python src/main.py. By default, it reads project settings from config/config.jsonc. 

python src/main.py

The repository assumes project-root paths for config/, data/, models/, and unit_tests/, so the default execution behavior stays anchored to the repository structure.
